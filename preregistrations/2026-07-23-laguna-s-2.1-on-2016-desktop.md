# Pre-registration: Laguna S 2.1 (118B-A8B) on the 2016 desktop

**Author:** Federico Sciuca ([quantprobe](https://github.com/FedericoTs/quantprobe))
**Date staked:** 2026-07-23 — committed BEFORE the model finished downloading; the git timestamp is the proof
**Status:** OPEN — measurement follows within days

## The claim

[Laguna S 2.1](https://huggingface.co/poolside/Laguna-S-2.1) (117.6B total, ~8B active MoE, released 2026-07-21) will run on my 2016 desktop
— i5-7600K, GTX 1060 6 GB, 16 GB DDR4-3000, **SATA** MX500 (0.45 GB/s measured) — as
[unsloth's UD-Q2_K_XL GGUF](https://huggingface.co/unsloth/Laguna-S-2.1-GGUF) (39.7 GB, ~2.7 effective bits),
experts streamed from disk, at

> **0.2–0.4 tok/s single-stream decode (midpoint 0.3)**

per the tiered decode law (`quantprobe plan --total 117.6 --active 8 --always-active 2.5 --bits 2.5 --machine 2016-xmp`
→ "0.3 tok/s — stream from disk (cold experts)"). Same genre as the GLM-4.5-Air run (110B-A12B: predicted
0.2–0.3, measured 0.19) — Laguna's smaller active set (8B vs 12B) should land it at or slightly above GLM-Air's speed.

Secondary (upgrade ladder, if/when hardware exists): NVMe ~3.5 GB/s → **~1.8 tok/s**; NVMe + 32 GB RAM → **~2.6**.

## Deliberate configuration choices

- **No DFlash draft.** Poolside ships a speculative-decoding draft model; per my P5 pre-registration
  (speculation × MoE antagonism on bandwidth-bound tiers — measured 2.3× slower on this box), running
  draftless is the law-optimal configuration here. If someone runs DFlash on a disk-bound setup and it
  *helps*, that refutes P5's scope.
- **Q2-class, not the IQ1 variants** (33.8 GB would fit easier): Law 2 says ~2-bit is the data-free floor
  and 1-bit collapses. The extra 6 GB buys a model that means something.
- Runtime: mainline llama.cpp **b10094** (Laguna support merged 2026-07-22, PR #25165) — not the vendor fork.

## Refuted if

Measured single-stream decode lands outside **0.15–0.5 tok/s** (band + measurement slack), or the model
fails to run at all on mainline b10094 with `-ot exps=CPU`-style streaming placement. Misses get published
with the same prominence as hits, per house rules.
